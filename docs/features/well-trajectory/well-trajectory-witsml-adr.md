# ADR: импорт WITSML (черновик)

**Статус:** черновик (фаза 4b)  
**Фаза 4a:** эндпоинты возвращают HTTP 501 с понятным сообщением.

## Контекст

Atlas Grid должен импортировать инклинометрию из корпоративных WITSML-хранилищ (версии 1.4 и 2.0).
Фаза 4a реализует CSV и Landmark `.wbp`; WITSML отложен до проверки выбора библиотеки.

## Варианты

| Библиотека | Версии WITSML | Примечания |
|------------|---------------|------------|
| `witsml` (community) | 1.4.x | Зрелые Python-биндинги; тяжёлый XML |
| `energistics` / vendor SDK | 1.4 / 2.0 | Зависит от корпоративного стека |
| Custom lxml + XSD | 1.4 / 2.0 | Полный контроль, выше стоимость сопровождения |

## Рекомендация (TBD)

1. Прототип объектов WITSML 1.4 `trajectory` / `log` через `witsml` или lxml на тестовом сервере.
2. Маппинг станций в `ImportParseResponse` planner (тот же формат, что у CSV / `.wbp`).
3. Повторное использование merge-пути backend `import_service` — без дублирования логики хранения.

## Вне scope (4a)

- Настройка OAuth / SOAP-клиента
- Массовый WITSML-импорт нескольких скважин по кустам
- Экспорт эллипсоидов погрешностей (фаза 4c)
