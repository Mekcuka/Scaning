# Профиль высот линий

Для **линейных** объектов на карте (дороги, трубопроводы, ЛЭП и т.д.) можно построить **высотный профиль** — таблицу отметок рельефа через равные шаги вдоль линии.

Исключение: линия **забоя ГС** (`well_bottomhole_gs`) — для неё вкладка «Профиль» не показывается.

Техническая спецификация: [line-elevation-profile.md](../../features/line-elevation-profile/line-elevation-profile.md).

## Как рассчитать

### По всему проекту

1. Откройте **Карту** проекта.
2. На панели инструментов откройте меню **Расчёт**.
3. Нажмите **Рассчитать профиль**.

Будут обработаны все линии проекта (кроме забоя ГС). При большом проекте расчёт может уйти в **фоновую задачу** — смотрите **журнал задач** в шапке (подпись «Профиль высот линий»).

### Для одной линии

1. Кликните линию на карте.
2. Откройте вкладку **Профиль**.
3. При необходимости задайте **шаг сэмплинга** (10–1000 м, по умолчанию 100) и **сохраните** объект.
4. Нажмите **Рассчитать профиль** (пересчитывает профили **всех** линий проекта, не только выбранной).

## Что показывает вкладка «Профиль»

После расчёта:

- длина линии и число точек;
- время расчёта и источник ЦМР (например `opentopography:COP30`);
- таблица: **пикет**, расстояние от начала (м), **отметка** (м), **уклон** (‰);
- переключатель **Таблица / График**;
- кнопка **Excel** на таблице — выгрузка в файл.

Если профиль ещё не считали — заглушка «Профиль не рассчитан».

## Профиль в 3D

На **3D-карте** линия с рассчитанным профилем повторяет **рельеф из таблицы** (отметки ЦМР), а не «плоский» MapLibre terrain. Для **ЛЭП** опоры стоят на отметках профиля, провода — выше. Линия без профиля ведёт себя как раньше. После расчёта профиля переключитесь в 3D или обновите вид — геометрия подтянется автоматически.

При расчёте профиля у **точечных** объектов (кроме забоя ГС) в карточке на вкладке **3D на карте** обновляется поле **«Отметка основания»** — абсолютная отметка рельефа ЦМР в координатах объекта.

## ЦМР на проект

В отличие от земляных работ площадки (ЦМР **на каждый куст**), здесь хранится **один** цифровой рельеф на весь проект. Граница выгрузки строится по всем объектам карты (кроме забоев). Если геометрия проекта сильно расширилась, при следующем расчёте ЦМР может загрузиться заново.

## Типичные ошибки

| Сообщение / ситуация | Что делать |
|----------------------|------------|
| Нет линий для расчёта | Добавьте линейные объекты на карту |
| Нет объектов для BBOX | В проекте нет инфраструктуры (кроме забоев) для охвата ЦМР |
| DEM не настроен на сервере | В production: `OPENTOPOGRAPHY_API_KEY`. В local dev без ключа при `ENVIRONMENT=development` используется синтетический ЦМР (отметки ~100 м); для staging задайте `DEM_ALLOW_SYNTHETIC=true` |
| Задача в журнале «Ошибка» | Раскройте карточку, скачайте JSON, проверьте `result` / `detail` |
