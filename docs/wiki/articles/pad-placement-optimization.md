# Оптимизация размещения кустов с карты

> **Статус функции:** ✅ реализовано (M1–M5, M2+ — перебор центра куста по Σ MD).  
> Подробная спецификация: [pad-placement-optimization.md](../../features/pad-placement/pad-placement-optimization.md).

Если на карте уже есть **забои** (куда бурить), а **кусты ещё не расставлены**, эта функция подскажет: **сколько** новых площадок нужно и **где** их поставить.

## Не путать с другими разделами

| Что | Зачем |
|-----|--------|
| **«Кустование»** (`/pad-clustering`) | Доработка **одного** уже созданного куста: устья, траектории, SF |
| **Число кустов в матрице POI** | Формула для **стоимости** («скважин ÷ скважин на КП»), **без координат** на карте |
| **Эта функция** | **Где на карте** поставить новые кусты по выбранным забоям |

## Что подготовить на карте

1. Нарисуйте **забои** инструментом **«Забой»**:
   - **ННБ** — одна точка = одна скважина;
   - **ГС** — сначала **Т1**, затем **Т3** (две точки = **одна** скважина).
2. В карточке забоя укажите **TVD** (глубину) и при необходимости угол и азимут.
3. Для **ГС** желательно задать координаты **Т1 и Т3** на карте — без них траектория не построится.
4. Привязка к уже существующему кусту **не обязательна** — расчёт создаёт **новые** площадки и не двигает старые.

### Сколько скважин участвует в расчёте

| На карте | В расчёте |
|----------|-----------|
| 2 забоя ННБ | 2 скважины |
| 1 ГС (Т1 + Т3) | 1 скважина |
| 2 ННБ + 1 ГС | 3 скважины (при этом **4 точки** на карте) |

## Пошаговая инструкция

1. Откройте **Карту** проекта.
2. Нажмите **«Кусты»** в панели инструментов (режим «Оптимизация кустов») и отметьте нужные **забои** (клик, рамка или «Видимые»).
3. В боковой панели задайте:
   - максимум скважин на одном кусте;
   - минимальное расстояние между новыми кустами;
   - шаг design, при необходимости — проверку SF.
4. В блоке **«Расширенные»** (по умолчанию включено):
   - **Оптимизировать положение куста (Σ MD)** — двухфазный перебор центра вокруг кластера забоев (сначала грубая сетка, затем точный расчёт для лучшей точки);
   - **Радиус поиска** и **шаг сетки** — точность vs время расчёта.
5. Нажмите **«Рассчитать»** (не путать с предпросмотром запроса — долгий этап именно здесь).
6. В таблице сравните **варианты**: число кустов, Σ MD, min SF, предупреждения.
7. Щёлкните по строке — на карте появится **предпросмотр** контуров и траекторий.
8. **«Применить»** — на карте появятся новые кусты, забои привяжутся к ним.

Если скважин **больше 8** или комбинаций разбиения много — расчёт автоматически уходит в **журнал задач** (иконка в шапке приложения). Таймаут одного запроса — до **10 минут**.

## Как читать таблицу результатов

| Колонка | Смысл |
|---------|--------|
| **Число кустов** | Сколько новых площадок — **меньше лучше** |
| **Σ MD** | Сумма измеренных глубин по стволам после расчёта траекторий — **меньше лучше** |
| **min SF** | Худшая пара SF (если включена проверка) |
| **Предупреждения** | Не удалось построить траекторию, слишком близко к существующему кусту и т.п. |

Сначала сравнивается **число кустов**. Если оно одинаковое — **Σ MD**, затем SF.

## Как система выбирает место куста

Для каждой группы скважин в варианте система:

1. Строит сетку точек (до **5×5**) вокруг **центра забоев**.
2. На каждой точке **грубо** оценивает Σ MD (укрупнённый перебор точки входа для ГС).
3. Выбирает лучший центр и делает **полный** расчёт траекторий только для него.

Отключить перебор можно в **«Расширенные»** (тогда используется упрощённая эвристика — один центр без сетки).

## Ускорить расчёт

| Действие | Эффект |
|----------|--------|
| Выключить **«Оптимизировать положение куста»** | Один прогон вместо ~26 |
| На забоях ГС: точка входа **Т1** или **Т3** (не «Любая») | Без перебора offset вдоль горизонтали |
| Увеличить **шаг сетки** (300–400 м) | Меньше точек в сетке |
| Отключить **проверку SF** | Без расчёта anti-collision на вариантах |
| Много скважин | Дождаться задачи в **журнале** (async) |

Проверка SF при оптимизации кустов **не заменяет** расчёт SF в «Кустовании» после применения.

## Что делать после применения

1. На карте появятся **нефтяные или газовые площадки**.
2. Забои будут связаны с соответствующими кустами.
3. Откройте **Кустование** для нужного куста:
   - уточните раскладку устьев и отметку устья (KB);
   - при необходимости пересчитайте траектории (там доступен полный перебор точки входа **с учётом SF**);
   - запустите **расчёт SF** (безопасные расстояния между стволами).

## Ограничения

- Уже нарисованные кусты **не перемещаются** — только **новые** площадки.
- Не заменяет расчёт числа КП в точке интереса для экономики.
- Рельеф (DEM) для отметки устья при оптимизации может не учитываться — уточняйте в «Кустовании» после apply.
- Большой радиус и малый шаг сетки **увеличивают время** расчёта.
