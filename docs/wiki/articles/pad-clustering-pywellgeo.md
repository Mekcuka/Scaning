## Вкладки раздела «Кустование»

Вверху страницы `/pad-clustering` три subnav-вкладки (как в «Параметрах»):

| Вкладка | Маршрут | Содержимое |
|---------|---------|------------|
| **Куст** | `/pad-clustering/workspace/{projectId}` | Sidebar «Куст / Расчёт / PyWellGeo» + 3D-сцена |
| **Сводка расчёта** | `/pad-clustering/summary/{projectId}` | Одна таблица **Раздел · Объект · Параметр · Значение** (read-only) |
| **Профиль траектории** | `/pad-clustering/profile/{projectId}` | График MD–TVD + таблица станций, Excel, маркеры Т1/Т3 (ГС) |

Старый URL `/pad-clustering/{projectId}` перенаправляет на **Куст**. Выбор куста, «Карта», «Сохранить» и (на профиле) селектор **Объект** — общие для всех subnav (`?padId=`, `?subject=`, `?well=`).

## Сводка расчёта

Read-only обзор последнего состояния куста без sidebar и 3D:

| Раздел таблицы | Что попадает в строки |
|----------------|----------------------|
| Куст | Имя, тип, KB, число устьев, дата расчёта траекторий, DEM |
| Параметры welleng | Шаг MD, модель погрешности, порог SF и др. из вкладки «Расчёт» |
| Скважины (основной ствол) | По одной группе строк на каждый рассчитанный ствол |
| Забои на кусте | NNB и ГС; доп.стволы — объект с префиксом `↳` |
| PyWellGeo | Метаданные деревьев по скважинам |

Если данных нет (например, не запускали «Рассчитать до забоев»), вместо блока — строка-подсказка.

## Профиль траектории

Две колонки на широком экране: **график MD–TVD** слева, **таблица станций** справа (на узком — друг под другом).

1. В шапке страницы выберите **Объект**:
   - `[Осн.] Скв-N` — профиль из welleng `survey.stations`;
   - `[Доп.] …` — main bore + боковая ветка PyWellGeo до выбранного доп.ствола.
2. На графике для **горизонтальных** забоев (ГС) отображаются **Т1** и **Т3** (начало и конец линии ГС на карте).
3. В таблице станций — MD, TVD, наклон, азимут, DLS, смещения N/E. Кнопка **Excel** в правом верхнем углу таблицы выгружает все строки выбранного объекта.

> **NNB** не имеет Т1/Т3 — маркеры только у ГС. Для доп.ствола-ГС Т1/Т3 берутся из забоя доп.ствола, а не основного.

## Зачем отдельная вкладка PyWellGeo

**PyWellGeo** дополняет расчёт **welleng** (вкладки «Куст» и «Расчёт»):

| welleng | PyWellGeo |
|---------|-----------|
| Design до **основного** забоя, ISCWSA clearance | Дерево скважины (ветви, перфорации) |
| Канон survey для SF | AHD, длины веток, thermal |
| Импорт `.wbp` / CSV | YAML (XYZGENERIC, DETAILEDTNO, DC1D) |

Survey welleng **не перезаписывается** автоматически при редактировании дерева.

## Основной забой и доп.ствол на карте

На **2D-карте** у забоя есть роль (не отдельный тип объекта):

| Роль | Назначение |
|------|------------|
| **Основной забой** | Цель основного ствола; «Рассчитать до забоев» на вкладке «Куст» |
| **Доп.ствол** | Дополнительная цель той же скважины; рисуется **фиолетовым**; привязан к **родительскому** основному забою |

Создание доп.ствола: **Карта → Забой → Доп.ствол (ННБ/ГС)** или поле «Роль» в карточке забоя.

На карте между основным и доп.стволом показывается **фиолетовый пунктир**, пока нет рассчитанной боковой ветки PyWellGeo. После расчёта «До забоя» пунктир **скрывается** — остаётся линия ветки на карте и в 3D «Кустования».

## Быстрый старт

1. На вкладке **Куст** выполните «Рассчитать до забоя» (welleng design) для **основных** забоев.
2. Откройте вкладку **PyWellGeo**.
3. Выберите скважину → **Импорт из survey welleng**.
4. Нажмите **Упростить ствол** (coarsen) — уменьшит число узлов для kick-off.
5. Выберите узел kick-off → **Добавить боковой ствол** (azim/dip, XYZ или **До забоя**).
6. Нажмите **Пересчитать** — на 3D появится слой **«Ветви»** (preview работает без «Сохранить»).
7. **Сохранить** на странице кустования — деревья пишутся в `pad_pywellgeo_trees_json`.

## Многозабойная скважина и доп.стволы

1. **Импорт из survey** — только основной ствол (welleng).
2. **Упростить ствол** — `coarsen` по шагу из настроек (default 75 м).
3. **Kick-off** — выберите узел на стволе (координаты x/y/z, z = −TVD).
4. **Боковой ствол** — PyWellGeo `add_xyz`:
   - азимут + dip + длина (ручной отрезок);
   - таблица XYZ (первая точка = kick-off на стволе);
   - **До забоя** — **welleng connector** от kick-off до целевого забоя. В списке целей:
     - **«Доп.стволы этой скважины»** — объекты с ролью доп.ствол, привязанные к основному забою текущего слота;
     - **«Доп.стволы других скважин куста»** / **«… других кустов»** — остальные lateral;
     - группы основных забоев — как раньше (NNB, ГС Т1/Т3).
   - Шаг дискретизации = `well_trajectory_step_m` куста (default 30 м); **DLS проектирования** (`dls_design`, default **3 °/30m**). Attitude kick-off — из survey основного ствола в точке узла. Это **не** полный GS horizontal от kick-off и **не** обновляет survey для SF.
5. **YAML** — экспорт/импорт XYZGENERIC с ключами `main_wellbore` + `lat1`, `lat2`, …

Clearance (SF) **не** пересчитывается по боковым стволам — канон welleng `survey.stations`. Забои **вне текущего куста** используются только как геометрическая цель PyWellGeo, не в `design-from-bottomholes`.

## Таблица забоев на «Кустовании»

На вкладке **Куст** в списке привязанных забоев:

- колонка **«Роль»** (Основной / Доп.ствол);
- доп.стволы показаны **с отступом** под своим основным забоем.

## Секции вкладки PyWellGeo

- **Скважина** — выбор `Скв-N`, импорт из survey, coarsen, apply geometry.
- **Дерево** — фильтр узлов, редактор, мастер бокового ствола.
- **YAML** — импорт/экспорт `.yml`.
- **Азимут / наклон** — калькулятор AzimDip (конвенция PyWellGeo ≠ welleng NDS без пересчёта).
- **Тепло и флюиды** — T(z), DC1D, свойства воды.
- **Настройки куста** — радиус по умолчанию, Tsurface, градиент.

## Данные в properties

|------|------------|
