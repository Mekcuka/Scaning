# Навигация по приложению

После входа открывается **боковое меню** (сайдбар). Разделы зависят от роли пользователя.

## Шапка приложения

Заголовок текущей страницы (например «Проекты», «Матрица решений», «Импорт данных») отображается в **глобальной шапке** (`app-header`), между меню и иконками AI-помощника, журнала задач и темы. Кнопки действий страницы (режимы таблицы, экспорт, выбор POI) остаются **в теле страницы**, не в шапке.

## Проект и контекст

Большинство экранов работают в контексте **активного проекта**. Его **UUID в конце адресной строки**:

```
http://localhost:5173/map/{id-проекта}
http://localhost:5173/matrix/{id-проекта}
http://localhost:5173/parameters/rates/{id-проекта}
```

При переходе по разделам сайдбара id проекта сохраняется. Смена проекта на дашборде или в списке проектов обновляет URL. Страница **Проекты → карточка проекта** (`/projects/{id}`) — настройки POI и анализ; быстрые ссылки «Карта», «Матрица» ведут на `/map/{id}`, `/matrix/{id}`.

**Без id проекта в URL:** вход (`/login`, `/register`), список проектов (`/projects`), карточка проекта (`/projects/{id}`), администрирование (`/admin/*`).

**Старые адреса** (`/map`, `/matrix`, `/{id}/map`, …) автоматически перенаправляются на `/раздел/{активный-проект}`.

## Основные разделы

| Раздел | Маршрут | Назначение |
|--------|---------|------------|
| Дашборд | `/dashboard/{projectId}` | Обзор |
| Проекты | `/projects` | Список проектов, создание (admin/analyst) |
| Карта | `/map/{projectId}` | 2D / **Площадки** / 3D — объекты и POI |
| Кустование | `/pad-clustering/workspace/{projectId}` (subnav: **Куст**, **Сводка**, **Профиль**) | Раскладка устьев, траектории, 3D куста, read-only сводка, профиль MD–TVD |
| Параметры | `/parameters/*/{projectId}` | Пропускная способность, **земляные работы**, **точки подключения**, песок, даты, **ставки** |
| Потоки | `/flows/*/{projectId}` | Технологическая, экономическая схемы, логистика песка |
| Матрица | `/matrix/{projectId}` | Сравнение POI по стоимости и кандидатам |
| Отчёты | `/report/{projectId}` | One-pager и печать |
| **Данные** | `/data/*/{projectId}` | Импорт, экспорт, импорт 3D (подвкладки) |
| Администрирование | `/admin/*` | Пользователи, журнал задач, параметры AI (только admin) |

### Подвкладки «Данные»

| Вкладка | Маршрут | Кто видит |
|---------|---------|-----------|
| Импорт | `/data/import/{projectId}` | admin, analyst, data_manager |
| Экспорт | `/data/export/{projectId}` | все роли |
| Импорт 3D | `/data/import-3d/{projectId}` | admin (загрузка); admin и владелец проекта (назначение) |

### Подвкладки «Кустование»

| Вкладка | Маршрут |
|---------|---------|
| Куст (workspace) | `/pad-clustering/workspace/{projectId}` |
| Сводка расчёта | `/pad-clustering/summary/{projectId}` |
| Профиль траектории | `/pad-clustering/profile/{projectId}` |

Legacy `/pad-clustering/{projectId}` → **Куст**.

Старые URL `/import`, `/export`, `/import-3d` и `/{projectId}/data/...` перенаправляются на `/data/.../{projectId}`.

## Выход

Кнопка **Выход** — в нижней части сайдбара (иконка).

## AI-помощник

Иконка помощника в **шапке** открывает чат. Он учитывает активный проект и текущую страницу.

**Администратор:** вкладка **Администрирование → AI-помощник** (`/admin/assistant`) — статус LLM и Wiki RAG, probe подключения, временный override (chat и embeddings) без рестарта сервера, справка по провайдерам и Cursor MCP. Постоянные ключи и URL — в `app.env` на сервере.
