# Параметры расчёта (справочник)

Краткое описание всех настроек `options` и их влияния на **SteinerPy** и **GeoSteiner**.

## Общие параметры (оба солвера)

### `connector_max_km` (по умолчанию 0.2)

Максимальная длина **листового ребра** от терминала до магистрали дерева (км).

- Учитывается вся цепочка: терминал → hub → attach → backbone.
- Пер-терминальный override: `terminals[].attachment_max_km`.
- При нарушении: вставляются `steiner:attach:*`, предупреждение `*_attachment_radius_violation`.

### `enforce_attachment_radius` (по умолчанию true)

- `true` — лимиты применяются.
- `false` — чистое Steiner-дерево без ограничений на длину примыкания.

### `normalize_terminal_leaves` (по умолчанию true)

Каждый терминал должен иметь степень 1. При необходимости вставляется промежуточный узел `steiner:hub:N` на ребре терминала.

### `steiner_hub_offset_km` (по умолчанию 0)

Длина ребра терминал → hub (км). При `0` hub совпадает с координатами терминала (нулевая длина первого сегмента).

### `edge_vertex_spacing_km` (по умолчанию 0)

Максимальная длина сегмента ребра (км). Длинные рёбра делятся узлами `steiner:waypoint:*`. Не влияет на топологию дерева, только на детализацию геометрии.

### `steiner_radius_km` (по умолчанию 0)

**Запретная зона** вокруг каждого терминала: настоящие Steiner-точки (`steiner:0`, …) не должны находиться ближе R.

| Солвер | Механизм |
|--------|----------|
| SteinerPy | Фильтрация кандидатов в графе + пост-обработка repel |
| GeoSteiner | Только пост-обработка repel |

Hub (`steiner:hub:*`) и attach (`steiner:attach:*`) **не отталкиваются** — это «мост» к терминалу.

При `steiner_radius_km > connector_max_km` возможен конфликт → предупреждение `*_steiner_radius_vs_attachment_conflict`.

---

## Только SteinerPy

### `attachment_angle_deg` (по умолчанию 90)

Целевой угол примыкания терминала к магистрали (градусы).

### `attachment_angle_penalty` (по умолчанию 0)

Вес штрафа за отклонение угла от `attachment_angle_deg`. При `0` штраф выключен. Чем больше значение, тем сильнее MIP «тянет» примыкания к целевому углу (обычно ~90°).

---

## Только GeoSteiner

### Связность при большом числе терминалов

GeoSteiner (`efst | bb`) иногда возвращает **несколько несвязных поддеревьев** с правильными Steiner-точками. Сервис автоматически:

1. Пересобирает дерево через SteinerPy с теми же Steiner-точками (если SteinerPy установлен).
2. Иначе добавляет кратчайшие «мосты» между компонентами.

Предупреждение: `geosteiner_disconnected_components_fixed`.

---

## Типы узлов в ответе

| ID | Цвет в прототипе | Назначение |
|----|------------------|------------|
| `steiner:N` | зелёный | Steiner-точка солвера |
| `steiner:hub:N` | жёлтый | Hub для листового терминала |
| `steiner:attach:N` | оранжевый | Точка примыкания в пределах лимита |
| `steiner:waypoint:N` | серый | Промежуточная вершина на длинном ребре |

---

## Рекомендуемые настройки для «плотной» сети

Как в типичном расчёте с hub/attach/waypoint:

```json
{
  "connector_max_km": 0.4,
  "enforce_attachment_radius": true,
  "normalize_terminal_leaves": true,
  "steiner_hub_offset_km": 0,
  "edge_vertex_spacing_km": 0.5,
  "steiner_radius_km": 0
}
```

Для предсказуемой связности при 13+ терминалах установите SteinerPy (`pip install steinerpy`) — он используется для склейки GeoSteiner-компонент.
