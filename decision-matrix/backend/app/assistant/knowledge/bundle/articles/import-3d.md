# Импорт 3D (custom GLB)

Раздел **Импорт 3D** (`/import-3d`) добавляет пользовательские 3D-модели для **точечных** объектов на карте в режиме **3D**. Линейные подтипы glTF не поддерживают.

## Кто что может

| Действие | Роли |
|----------|------|
| Загрузка `.glb` (до ~20 МБ) | **admin** |
| Назначение подтипам, превью, редактирование метаданных | **admin** или **владелец проекта** |
| Выбор custom-модели в карточке объекта | пользователь с правом записи на инфраструктуру |

Пункт меню виден, если есть права на upload или assign для активного проекта.

## Типичный сценарий

1. Выберите **активный проект** в шапке.
2. **Администратор:** в блоке «Загрузка GLB» выберите файл, укажите **высоту модели (м)**, нажмите «Загрузить на сервер».
3. В «Назначение подтипам» выберите модель и отметьте **один или несколько подтипов** (чекбоксы).
4. Опционально включите **«Применить к объектам на карте»**:
   - *Только объекты без модели* — проставит модель точкам без своего `render_3d_model_id`;
   - *Все объекты подтипа* — перезапишет модель у всех точек выбранных подтипов (с подтверждением).
5. Перед сохранением показывается **превью счётчика** (сколько объектов будет обновлено).
6. Проверьте результат в блоке **«Превью на карте»** (справа на широком экране, остаётся на месте при прокрутке) или откройте **Карту** → режим **3D**.

## Таблица «Модели проекта»

На широком экране таблица — **под формами в левой колонке**; превью — справа.

Колонки: имя, размер, высота, число использований, подтипы, дата загрузки.

- **Изменить** — имя и высота модели.
- **Удалить** — с предупреждением, если модель используется объектами.

## На карте

В карточке объекта → вкладка **3D** → **«Модель 3D»**: «Стандартная» (Kenney по подтипу) или одна из назначенных custom-моделей.

## Если модель не видна

- Жёсткое обновление страницы (**Ctrl+F5**).
- Проверьте назначение подтипов в Импорт 3D.
- Убедитесь, что на точке выбрана custom-модель (не только назначение подтипа).
- На production: перелогин при ошибках CSRF при upload.

Технические детали хранения на сервере — в документации deploy (volume `map3d_models`).
