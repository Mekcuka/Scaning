# Фоновые задачи

Тяжёлые операции (анализ POI, массовый расчёт, часть импортов, **профиль высот линий**, земляные работы с DEM) выполняются **в фоне**, чтобы не блокировать интерфейс.

## Где смотреть статус

- **Иконка журнала задач** в шапке приложения — задачи **текущего проекта** (до ~30 записей в сессии браузера).
- **Админ → Журнал задач** (`/admin/jobs`) — все задачи системы (только **admin**); по **10 записей на страницу**, новые сверху.

## Статусы

| Статус | Значение |
|--------|----------|
| pending | В очереди |
| running | Выполняется |
| completed | Успешно |
| failed | Ошибка (см. детали в журнале) |
| cancelled | Отменена пользователем |

## Отмена

Пользователь с правом записи может **отменить** активную задачу проекта. В админ-панели admin может отменять шире.

## AI-помощник

Спросите: «есть ли активная задача», «статус фоновой задачи» — помощник запросит данные через tools. Не называйте внутренние UUID задач, если не знаете их — помощник найдёт активную задачу проекта.

## Зависшие задачи

Сервер автоматически помечает «зависшие» задачи по таймауту, чтобы можно было запустить новый расчёт.

## Примеры типов задач

| Подпись в журнале | Когда запускается |
|-------------------|-------------------|
| Анализ окружения | Массовый анализ POI |
| Логистика песка | Расчёт на вкладке логистики |
| Профиль высот линий | **Рассчитать профиль** на карте |
| Импорт файла | Асинхронный импорт |

Подробнее о профиле линий: [line-elevation-profile.md](line-elevation-profile.md).
