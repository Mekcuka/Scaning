# Импорт данных

Раздел **Данные → Импорт** (`/data/import`) доступен ролям **admin**, **analyst**, **data_manager**. Viewer импорт не выполняет.

На странице три **карточки** (как на экспорте):

1. **Импорт файлов** — CSV, GeoJSON, KML, Shapefile (ZIP), в т.ч. проект **Искра (SPARK)**.
2. **Подключение API** — REST-источник с тестом и синхронизацией.
3. **Импорт инклинометрии** — траектории скважин на куст (CSV, `.wbp`).

Сверху — выбор **целевого проекта**; снизу — **история импорта** по проекту.

## Поддерживаемые форматы (файлы)

- CSV, GeoJSON, KML, Shapefile (zip)
- **Проект Искра (SPARK)** — специализированный импорт с маппингом слоёв

## Типичный сценарий

1. Откройте **Данные → Импорт** (`/data/import`).
2. Выберите **проект** в панели «Проект для импорта».
3. В нужной карточке загрузите файл или настройте API-подключение.
4. Для файлов — при необходимости **Превью (dry-run)** перед финальным импортом.
5. Дождитесь завершения — крупный импорт может идти **фоновой задачей** (прогресс на странице и в журнале задач в шапке).
6. Проверьте результат на **карте** и в **истории импорта** на той же странице.

## Искра (SPARK)

Импорт Искры переносит слои и объекты по правилам маппинга подтипов инфраструктуры. После импорта проверьте видимость слоёв в панели «Слои» на карте.

## Журнал

История импортов — таблица внизу страницы импорта; через API и AI-tool `list_import_logs`.

## Ошибки

При ошибке в журнале будет код и сообщение. Частые причины: неверная проекция координат, неизвестный подтип объекта, нет прав на проект.
